# UI UX 및 성능 에이전트 컴포넌트 개발 가이드

---

이 가이드는 MUtang 프로젝트의 공통 컴포넌트(src/components) 개발 시 성능과 접근성을 극대화하기 위해 에이전트가 준수해야 할 핵심 지침입니다.

---

[핵심 개발 원칙]

- 저사양 스마트폰 하드웨어 최적화:
모든 컴포넌트는 CPU 및 RAM의 순간 소모량을 최소화해야 합니다. 특히 바코드 스캐너와 같이 용량이 큰 라이브러리를 동반하는 컴포넌트는 dynamic import를 사용하여 실제 모달이 켜질 때만 다운로드하도록 비동기 설계합니다.

- Canvas 및 이미지 연산 지연 처리:
사용자의 터치 입력이 일어나는 동안에는 canvas.toDataURL() 등의 무거운 픽셀 인코딩 연산을 실시간으로 계속 실행하지 않으며, 터치가 완전히 끝나거나 최종 승인 버튼을 누른 시점에 단 한 번 수행되도록 제어합니다.

- 고대비 다크모드 및 다국어 지원 레이아웃:
가독성 등급 AAA를 준수하는 고대비 컬러를 유지하고, 언어 전환(영어, 타갈로그어, 한국어 등) 시 텍스트 길이가 크게 달라지거나 레이아웃 틀어짐 및 깨짐 현상이 없는지 모바일 가로 너비 기준(320px~480px)에서 사전에 검증합니다.

- CSS 기반 애니메이션 최적화:
애니메이션 처리가 필요한 컴포넌트에서는 framer-motion 등의 자바스크립트 스레드 기반 도구 사용을 지양하고, 브라우저 GPU 하드웨어 가속을 적극 활용하는 Tailwind CSS 4.x 기반 고유의 transition을 설계하여 프레임 드롭을 차단합니다.
