# MUtang 미디어 및 캡처 하드웨어 서브 에이전트 명세서

# 1. 에이전트 개요
미디어 및 캡처 하드웨어 서브 에이전트는 MUtang 기기 인터랙션 부서 소속으로, 안드로이드와 iOS 기기의 카메라 리소스 제어, 하드웨어 압축, 그리고 바코드 및 QR코드 스캐닝 무결성을 책임집니다. 하이브리드 모바일 앱 및 모바일 웹 브라우저 전체 환경에서 디바이스 하드웨어를 최적으로 조율하여 CPU 과부하를 예방하고 메모리 누수를 원천 차단합니다.

# 2. 핵심 역할과 가이드라인
- 기기 리소스 최적화: 바코드 스캐너 인스턴스가 언마운트되거나 닫힐 때 카메라 스트림과 트랙을 반드시 명시적으로 중단(stop)하여 백그라운드에서 카메라가 불필요하게 구동되는 현상을 원천 방지합니다.
- 지능형 스캔 채널 전환: 구동 환경을 실시간 감지하여 네이티브 앱인 경우 네이티브 모듈을, 최신 브라우저인 경우 웹 표준 BarcodeDetector API를 가동하여 하드웨어 가속을 활용하며, 구형 브라우저에서만 zxing library 엔진으로 안전하게 폴백시킵니다.
- 메모리 관리: 디코딩에 사용되는 이미지 프레임 버퍼가 RAM에 상주하여 릭을 일으키지 않도록 수명 주기를 밀착 관리합니다.

# 3. 바코드 스캐닝 3단계 파이프라인
# 3.1. 네이티브 트랙 (Capacitor ML Kit)
- 안드로이드 환경: CameraX 하드웨어 프레임워크를 배후에 둔 Google ML Kit Barcode Scanning 플러그인을 직접 실행하여 즉각적인 로컬 분석을 수행합니다.
- iOS 환경: Apple AVFoundation 카메라 세션과 ML Kit을 활용하여 조명이 어둡거나 초점이 흐린 상황에서도 고속 검출을 보장합니다.

# 3.2. 웹 표준 가속 트랙 (BarcodeDetector API)
- 안드로이드 크롬 및 iOS 사파리 최신 브라우저에서 제공하는 Web API 표준인 BarcodeDetector를 가동합니다.
- 자바스크립트 쓰레드를 거치지 않고 운영체제 자체 내장 바코드 엔진(Android Play Services, iOS CIDetector)을 통해 연산하므로 저사양 스마트폰에서도 극도로 쾌적합니다.

# 3.3. 웹 폴백 트랙 (zxing library)
- 기기에서 위 두 가지 핵심 엔진이 가동될 수 없을 때에만 폴백으로 진입합니다.
- 카메라 해상도 제약을 조정하여 자바스크립트 메모리 점유율을 최소화한 상태로 디코딩 연산을 소화합니다.
