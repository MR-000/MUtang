# 모바일 클라이언트 및 기기 성능 최적화 가이드

이 문서는 모바일 화면 및 저사양 스마트폰 렌더링 성능을 최적화하기 위해 준수해야 할 개발 지침입니다.

[1. 모바일 저사양 및 접근성 수칙]

- 터치 영역 확보:
모든 터치 가능한 요소는 최소 44px 이상의 충분한 크기를 보장하여 오터치를 예방합니다.

- 렌더링 성능 보호:
캔버스를 그리거나 터치 이벤트를 처리할 때 연산 빈도를 조절하여 저사양 스마트폰에서도 렉이 걸리지 않도록 방어합니다.

- 지연 로딩 도입:
무거운 모듈이나 이미지 등은 동적 로딩을 적용해 첫 페이지 로딩 속도를 향상시킵니다.


[2. 미디어 및 캡처 하드웨어 제어 수칙]

- 사전 압축 전처리:
GCash 스크린샷이나 영수증 이미지를 업로드할 때, 클라이언트 측에서 미리 압축을 수행하여 전송 데이터 크기를 줄이고 서버 부하를 방지합니다.

- 하드웨어 자원 회수:
카메라 캡처나 스캐너 기능을 사용할 때 스캐너 창이 닫히면 스트림을 반드시 정지시켜 메모리 누수가 발생하지 않도록 감시합니다.
