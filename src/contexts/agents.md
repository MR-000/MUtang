# 글로벌 비즈니스 에이전트 컨텍스트 상태 가이드

---

이 가이드는 MUtang 프로젝트의 전역 상태 관리 모듈(src/contexts)을 고도화할 때, 성능 저하와 잦은 전역 리렌더링을 차단하기 위해 준수해야 할 핵심 지침입니다.

---

[핵심 개발 원칙]

- 테마 및 언어 설정의 클라이언트 동기화:
AuthContext.tsx에서 다국어 설정(Language)과 테마(Theme)를 통합하여 관리하며, 사용자 설정이 로컬 스토리지에 즉시 동기화되도록 설계합니다. 초기 렌더링 시 서버 사이드와 클라이언트 사이드의 상태 불일치(Hydration Mismatch) 현상이 일어나지 않도록 useEffect 내에서 안전하게 동기화합니다.

- 불필요한 전역 리렌더링 통제:
AuthContext가 제공하는 사용자 프로필 정보가 업데이트될 때, 이와 무관한 단순 UI 컴포넌트까지 전역 리렌더링의 영향을 받지 않도록 상태 구조를 조율합니다.

- Supabase Auth 상태 전환 속도 최적화:
onAuthStateChange 이벤트를 감지할 때, 인증 세션이 변동될 때만 한정하여 프로필 데이터를 갱신(fetchProfile)함으로써 데이터베이스 네트워크 요청 빈도를 효율적으로 절감합니다.
