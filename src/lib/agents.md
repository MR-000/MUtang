# 글로벌 비즈니스 및 인프라 에이전트 라이브러리 가이드

---

이 가이드는 MUtang 프로젝트의 유틸리티 및 API 연동 모듈(src/lib) 개발 시 데이터 정합성과 연산 성능을 보장하기 위해 준수해야 할 핵심 지침입니다.

---

[핵심 개발 원칙]

- 글로벌 다국어 시스템 일관성:
i18n.ts를 통해 관리되는 5개 국어 번역이 비즈니스 용어와 법적 고지 문구 상에서 완전히 일관성을 유지하도록 설계합니다. 번역 키가 누락되거나 영문이 폴백으로 대체될 때 사용자 경험을 헤치지 않도록 방어 로직을 수립합니다.

- 연산 오프로딩 및 백그라운드 처리:
kyc.ts 내의 신분증 화질 판별 루프와 같은 고부하 픽셀 분석 연산은 모바일 브라우저의 UI 메인 스레드를 장악하지 않도록 Web Worker를 도입해 백그라운드로 격리합니다. 또한 업로드 전 원본 카메라 이미지를 1280px 이하로 압축하여 대역폭 낭비를 차단합니다.

- 결제 시스템 중복 승인 원천 방지:
GCash SMS 알림 데이터와 Solana 토큰 전송 내역 처리 시, 각각의 참조 번호(Ref No) 및 블록체인 거래 영수증 키(TxID)를 데이터베이스 레벨에서 고유(UNIQUE)하게 관리하여 중복 호출 시 절대 중복 크레딧 지급이 발생하지 않도록 트랜잭션 예외 처리를 엄격히 제어합니다.

- 실시간 환율 오차 조율:
USDT, USDC 등 달러 가치 기반의 자산을 페소 크레딧으로 전환할 때 실시간 환율 API와의 안정적인 동기화 및 API 장애 시 폴백(Fallback) 변환 계수가 즉시 반영되도록 구현합니다.
